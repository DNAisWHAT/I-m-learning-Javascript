객체는 세 종류가 있다.   
1. 표준 빌트인 객체 : Object, String, Math 같은 룰 그 자체. 얘네들은 공식적으로 JS에서 사용하는 객체다. 전역 객체의 프로퍼티로 제공되므로 전역변수처럼 사용할 수 있다. 전역 객체 이름은 생략할 수 있기 때문이다.   
2. 호스트 객체 : 추가적인 객체, Node.js나 크롬같은 브라우저에서 제공해주는 객체이다.   
3. 사용자 정의 객체 : 뭐 내가 대충 생성자 함수로 만들 수 있는 객체들 아닐까?   
빌트인 객체는 전역 객체의 소유다.   
데이터 타입을 가진 변수가 메서드를 사용하는 이유는 변수가 객체로 순간 변했기 때문이다. 데이터가 .문법(객체 문법)을 사용하려고 할 때 래퍼객체라고 하는 객체가 데이터를 내부 슬롯에 담고 데이터 타입 프로토타입을 프로토타입으로 가져 프로토타입 메서드를 사용할 수 있게 된다.   
```javascript
const str = 'test';
console.log(str.length); // . 까지 읽고 래퍼 객체 생성 -> 프로토타입인 String.prototype 의 프로토타입 메서드인 length를 사용한다. 이 문 이후 참조하지 않으므로 래퍼객체가 삭제된다. 
```
선언되지 않은 변수를 함수에서 참조하고 있을 경우 에러가 발생하지 않고 그 값은 전역 객체의 프로퍼티로 들어간다. 이를 암묵적 전역이라고 한다. delete 키워드로 삭제 가능하다.
